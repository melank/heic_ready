# リリースノートテンプレート

以下をコピーして、リリースごとに内容を埋める。

```md
# HEIC Ready vX.Y.Z

Release date: YYYY-MM-DD

## Added

- 

## Changed

- 

## Fixed

- 

## Known Issues

- 

## Upgrade Notes

- 
```

## 運用ルール

- `Added / Changed / Fixed` はユーザー影響のある項目を優先して記載する
- 内部実装のみの変更は必要に応じて `Changed` に簡潔に記載する
- `Known Issues` が空でも、該当なしであることを明示する
- リリース本文はこのテンプレート構成を維持する
