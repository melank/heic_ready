# ランディングページ品質ガイド

このドキュメントは、`site/` 配下のランディングページ品質を一定に保つための基準を定義する。

## 対象

- `site/index.html`
- `site/styles.css`

## 品質基準

## 1. メッセージ

- ファーストビューで「何を解決するツールか」を1文で説明する
- CTA（ダウンロード導線）は明確に1つ以上配置する
- 誇張表現は避け、実装済み機能のみ記載する

## 2. デザイン

- PC/モバイルの両方でレイアウトが崩れない
- 見出し・本文・補助情報で視覚階層を分ける
- 余白/タイポグラフィ/コントラストを意図的に設計する

## 3. アクセシビリティ

- `lang` 属性を適切に設定する
- リンクやボタンは文脈なしで意味が分かる文言にする
- 装飾要素には `aria-hidden` を適用する
- 文字色と背景色は十分なコントラストを確保する

## 4. パフォーマンス

- 画像・フォントの外部依存を最小化する
- 初期表示に不要なスクリプトを追加しない
- 静的HTML/CSSで実現できる内容はJSに逃がさない

## 5. 運用

- 変更時は以下を確認する:
  1. `python3 -m http.server 8080 -d site` で表示確認
  2. モバイル幅（<= 820px）で表示確認
  3. CTAリンクが期待先へ遷移することを確認
- ページ構成や導線を変えた場合は `docs/github-pages.md` を更新する

## レビュー時チェックリスト

- 目的と導線が5秒以内に理解できるか
- 主要情報（機能・取得方法・リリース導線）が不足していないか
- デザイン変更が既存トーンと矛盾していないか
- 無関係な技術情報をトップページに詰め込みすぎていないか
